# CRM Data Quality Entity Contracts (Eng-5)

## Contract header
- Contract family: `crm-dq-canonical`
- Contract version: `v1.0`
- Compatibility rule: additive-only changes are minor (`v1.x`), removals/type changes require major bump (`v2.0`) per ADR-005.
- Serialization default: JSON objects, UTC timestamps in RFC3339 format.

## Eng-2 alignment summary
- `extract` contract output: `DQ_SCAN_RUN`, `DQ_ENTITY_SNAPSHOT`
- `score` contract output: `DQ_RULE_RESULT`, `DQ_SCORE_SUMMARY`, optional `DQ_REMEDIATION_ACTION`
- `publish` contract output: `DQ_PUBLISH_EVENT`

## DQ_SCAN_RUN
Purpose: top-level execution envelope for one extraction/scoring/publish cycle.

| Field | Type | Required | Constraints | Mapping notes |
| --- | --- | --- | --- | --- |
| `scan_run_id` | string (uuid) | yes | non-empty, unique per run | Generated by orchestration layer |
| `contract_version` | string | yes | regex `^v[0-9]+\.[0-9]+$` | Must match canonical contract set |
| `source_system` | enum | yes | `dataverse` | Fixed in current scope |
| `source_env` | string | yes | non-empty | Dev/Test/Prod environment label |
| `started_at_utc` | datetime | yes | RFC3339 UTC | Start time of extract |
| `completed_at_utc` | datetime | no | `>= started_at_utc` when set | Set when run reaches terminal state |
| `status` | enum | yes | `running`,`completed`,`failed`,`cancelled` | Run lifecycle state |
| `extracted_entity_count` | integer | yes | `>= 0` | Count of snapshots produced |
| `evaluated_rule_count` | integer | yes | `>= 0` | Count of rule evaluations |
| `overall_score` | number | no | `0..100` | Present at/after scoring |
| `correlation_id` | string | yes | non-empty | Trace id for observability contracts |

Validation rules:
- Reject if `completed_at_utc` is present and `status=running`.
- Reject if `status=completed` and `overall_score` is null.

## DQ_ENTITY_SNAPSHOT
Purpose: normalized entity state captured from Dataverse for scoring.

| Field | Type | Required | Constraints | Mapping notes |
| --- | --- | --- | --- | --- |
| `snapshot_id` | string (uuid) | yes | unique | Internal surrogate key |
| `scan_run_id` | string (uuid) | yes | FK to `DQ_SCAN_RUN.scan_run_id` | Run ownership |
| `entity_logical_name` | enum | yes | `account`,`contact`,`lead`,`opportunity` | Initial ADR-001 bounded entities |
| `entity_primary_id` | string | yes | non-empty | Dataverse record id (guid) |
| `entity_primary_name` | string | no | max 300 chars | Typically `name`/fullname; tenant-specific |
| `region_code` | string | no | max 32 chars | Requires environment metadata mapping |
| `owner_principal_id` | string | no | guid format | Dataverse owner/user/team id |
| `modified_on_utc` | datetime | no | RFC3339 UTC | Source last-modified timestamp |
| `ingest_hash` | string | yes | SHA-256 hex | Snapshot dedupe/integrity |
| `pii_policy_tag` | enum | yes | `none`,`masked`,`restricted` | Field-policy alignment (ADR-008) |

Validation rules:
- Reject if `entity_logical_name` outside bounded set.
- Reject duplicate pair (`scan_run_id`,`entity_logical_name`,`entity_primary_id`).

## DQ_RULE_RESULT
Purpose: atomic outcome of one quality rule against one snapshot.

| Field | Type | Required | Constraints | Mapping notes |
| --- | --- | --- | --- | --- |
| `rule_result_id` | string (uuid) | yes | unique | Generated at evaluation |
| `scan_run_id` | string (uuid) | yes | FK to `DQ_SCAN_RUN` | Supports run-level aggregation |
| `snapshot_id` | string (uuid) | yes | FK to `DQ_ENTITY_SNAPSHOT` | Entity scope |
| `rule_id` | string | yes | non-empty | Rule catalog identifier |
| `rule_version` | string | yes | semver-ish string | Rule lifecycle support |
| `severity` | enum | yes | `critical`,`high`,`medium`,`low` | Drives prioritization |
| `status` | enum | yes | `pass`,`warn`,`fail`,`skip` | Outcome state |
| `score_impact` | number | yes | `-100..100` | Contribution to overall score |
| `reason_code` | string | yes | non-empty | Deterministic failure category |
| `message` | string | no | max 1000 chars | Human-readable details |
| `evaluated_at_utc` | datetime | yes | RFC3339 UTC | Evaluation timestamp |

Validation rules:
- Reject if `status=pass` and `score_impact < 0`.
- Reject if `status=fail` and `severity=low` for critical-only rules (rule metadata check).

## DQ_SCORE_SUMMARY
Purpose: aggregated scoring output for a scan run.

| Field | Type | Required | Constraints | Mapping notes |
| --- | --- | --- | --- | --- |
| `score_summary_id` | string (uuid) | yes | unique | One summary per run |
| `scan_run_id` | string (uuid) | yes | unique FK to `DQ_SCAN_RUN` | One-to-one with run |
| `overall_score` | number | yes | `0..100` | Weighted aggregate |
| `completeness_score` | number | yes | `0..100` | Sub-metric |
| `consistency_score` | number | yes | `0..100` | Sub-metric |
| `validity_score` | number | yes | `0..100` | Sub-metric |
| `fail_count` | integer | yes | `>= 0` | Count of failed rules |
| `warn_count` | integer | yes | `>= 0` | Count of warned rules |
| `pass_count` | integer | yes | `>= 0` | Count of passed rules |
| `score_band` | enum | yes | `excellent`,`good`,`fair`,`poor`,`critical` | Threshold-based label |

Validation rules:
- Reject when `fail_count + warn_count + pass_count = 0` and run is completed.
- Reject when `overall_score` is outside computed band threshold table.

## DQ_REMEDIATION_ACTION
Purpose: recommended corrective action tied to a rule failure/warn.

| Field | Type | Required | Constraints | Mapping notes |
| --- | --- | --- | --- | --- |
| `remediation_id` | string (uuid) | yes | unique | Generated action id |
| `scan_run_id` | string (uuid) | yes | FK to `DQ_SCAN_RUN` | Run context |
| `snapshot_id` | string (uuid) | yes | FK to `DQ_ENTITY_SNAPSHOT` | Entity context |
| `rule_result_id` | string (uuid) | yes | FK to `DQ_RULE_RESULT` | Source outcome |
| `action_type` | enum | yes | `update_field`,`merge_duplicate`,`enrich`,`manual_review` | Action class |
| `target_field` | string | no | max 128 chars | Required for `update_field` |
| `suggested_value` | string | no | max 1000 chars | Optional recommendation |
| `confidence_band` | enum | yes | `high`,`medium`,`low` | Recommendation confidence |
| `priority` | enum | yes | `p0`,`p1`,`p2`,`p3` | Work queue priority |
| `created_at_utc` | datetime | yes | RFC3339 UTC | Action creation time |

Validation rules:
- Reject if `action_type=update_field` and `target_field` is null/empty.
- Reject if linked `rule_result.status=pass`.

## DQ_PUBLISH_EVENT
Purpose: persisted publish/write-back attempt record.

| Field | Type | Required | Constraints | Mapping notes |
| --- | --- | --- | --- | --- |
| `publish_event_id` | string (uuid) | yes | unique | Event id |
| `scan_run_id` | string (uuid) | yes | FK to `DQ_SCAN_RUN` | Run context |
| `snapshot_id` | string (uuid) | no | FK to `DQ_ENTITY_SNAPSHOT` | Optional per-record granularity |
| `destination` | enum | yes | `dataverse`,`lakehouse`,`queue` | Initial publish targets |
| `destination_record_id` | string | no | non-empty when success | Dataverse write-back id |
| `publish_status` | enum | yes | `pending`,`succeeded`,`failed`,`dead_lettered` | Publish state |
| `attempt_count` | integer | yes | `>= 1` | Retry counter |
| `published_at_utc` | datetime | no | RFC3339 UTC | Set on terminal attempt |
| `error_code` | string | no | max 128 chars | Present on failures |
| `error_message` | string | no | max 1000 chars | Sanitized failure detail |

Validation rules:
- Reject if `publish_status=succeeded` and `destination_record_id` is empty for `dataverse` destination.
- Reject if `publish_status=failed` and `error_code` is empty.

## Cross-entity invariants (unit-test seed rules)
- Every entity except `DQ_SCAN_RUN` must reference an existing `scan_run_id`.
- `DQ_RULE_RESULT.snapshot_id` must belong to the same `scan_run_id` as its row.
- Once a `DQ_SCAN_RUN.status` is `completed` or `failed`, child rows are append-only and immutable.
- `contract_version` mismatch between parent and child entities is invalid.

## Explicit assumptions pending live Dataverse confirmation
- Logical names are constrained to `account/contact/lead/opportunity` for `v1.0`; additional entities require contract minor update.
- `owner_principal_id` type and resolver behavior may differ between user/team-owned rows.
- Region mapping source may come from business unit or custom fields and must be validated in Eng-6 live handshake.
