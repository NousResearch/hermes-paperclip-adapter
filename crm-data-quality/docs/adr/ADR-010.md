# ADR-010: Observability and Operational Readiness Baseline

## Status
Accepted

## Context
The integration must be supportable from day one. Without minimum telemetry standards, failures in auth, sync, or scoring are hard to isolate quickly.

## Decision
Define baseline observability for v1: structured logs, correlation IDs across extract/score/publish stages, and core metrics (latency, throughput, failure classes, retry counts).

## Consequences
- Positive: Faster incident triage and safer rollout decisions.
- Positive: Enables objective SLO tracking for integration reliability.
- Negative: Additional implementation and dashboard setup effort.
- Negative: Telemetry schema drift must be governed.

## Alternatives Considered
- Best-effort logging only: rejected due to weak diagnostics.
- Heavy observability platform integration in v1: rejected to keep initial scope lean.
