# ADR-003: Authentication Flow for Dataverse Access

## Status
Accepted

## Context
The service needs secure, non-interactive access to Dataverse for scheduled and on-demand quality runs. User-delegated tokens are operationally fragile for backend workloads.

## Decision
Use Azure AD application identity with client credentials flow and least-privilege app roles for Dataverse API access.

## Consequences
- Positive: Stable non-interactive auth for automation.
- Positive: Easier rotation and policy enforcement at tenant level.
- Negative: Initial app registration and consent setup required.
- Negative: Mis-scoped roles can silently limit data visibility.

## Alternatives Considered
- Delegated user auth: rejected due to token lifecycle and user-dependency risk.
- Shared service account username/password: rejected for security and compliance reasons.
