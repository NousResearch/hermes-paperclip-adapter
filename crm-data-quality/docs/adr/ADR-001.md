# ADR-001: Canonical CRM Entity Scope for Data Quality

## Status
Accepted

## Context
CRM data quality checks can expand across many Dataverse tables, but broad scope early increases mapping complexity and operational risk. The initial gate requires explicit entity selection for integration.

## Decision
Limit initial data-quality integration scope to three canonical entities: `account`, `contact`, and `lead`.

## Consequences
- Positive: Faster delivery of usable checks with predictable schema coverage.
- Positive: Clear ownership for mapping, validation, and remediation workflows.
- Negative: Opportunities and custom entities are deferred.
- Negative: Some downstream use cases will need phase-two extension.

## Alternatives Considered
- Include all sales entities in v1: rejected due to high schema variance and elevated rollout risk.
- Start with only `account`: rejected because cross-entity quality signals (account-contact-lead relationships) are core to CRM DQ outcomes.
