# ADR-006: Rule Execution Boundary and Extensibility

## Status
Accepted

## Context
Data-quality rules differ by entity and region. Embedding rules directly in transport code makes iterative tuning high risk and slow.

## Decision
Separate rule execution into a pluggable rules layer driven by declarative rule definitions and a stable evaluation interface.

## Consequences
- Positive: Faster rule updates without transport rewrites.
- Positive: Better testability of rule behavior.
- Negative: Requires definition validation and lifecycle controls.
- Negative: Debugging can span rule config plus runtime engine.

## Alternatives Considered
- Hard-code rules in integration service: rejected for maintainability.
- External third-party rule engine in v1: rejected to limit dependency and operational complexity.
